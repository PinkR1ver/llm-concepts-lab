## Design Context

### Users
- Primary user: Jude, using this as a personal LLM learning lab
- Secondary audience: other people who may view it publicly as an educational concept site
- Primary jobs to be done:
  - understand LLM concepts through code and visualization
  - inspect how transformations happen step by step
  - revisit concepts through a structured, navigable interface

### Brand Personality
- digital exhibition
- refined developer tool
- concept-first, precise, exploratory
- the interface should feel thoughtful, deliberate, and slightly gallery-like rather than like a generic dashboard

### Aesthetic Direction
- a hybrid of digital exhibition and polished developer tool
- should feel suitable both for personal study and public sharing
- needs both dark mode and light mode
- avoid generic AI UI aesthetics and avoid admin-dashboard blandness
- visual tone should be elegant, restrained, inspectable, and memorable

### Design Principles
1. Every surface should help explain a concept, not merely decorate it.
2. The app shell should support long-term growth across many concept modules.
3. Navigation should feel like moving through an exhibition of mechanisms, not a pile of pages.
4. Code, matrices, tokens, and explanations should feel precise and tactile.
5. Visual polish should serve clarity first, but still leave a strong aesthetic impression.
